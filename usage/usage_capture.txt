Usage:

  vdb capture [OPTIONS] PORT [FILE]

Options:

  -6 --ipv6                 Use IPv6
  -a --address=ADDR         Use IP address (required for playback only)
  -n --iface=IFACE          Use network interface, e.g. 'eth0'
  -c --color-off            Do not use colored output
  -d --dump                 Print PDU hex dump (overrides -s and -x)
  -e --exercise=E           Captures only PDUs on exercise number E
  -t --pdu-type=T1,...      Captures only PDUs with PDU type T (numeric)
  -f --pdu-family=F1,...    Captures only PDUs with PDU family F (numeric)
  -i --ids=I1,...           Captures only PDUs with specified entity ID(s)
  -h --help                 Print this info
  -x --extract              Print PDU data extraction (overrides -s)
  -q --quiet                Print as little as possible
  -s --summarize            Print short PDU summary
  -v --verbose              Print as much as possible (debugging)
  -w --warnings             Show warnings (suppressed by default)
  -E --errors-off           Do not show any error messages
  
Examples:

  Captures VDIS traffic on port 3000 from IP 192.168.255.255 without saving 
  PDUs to file:
  
    $ vdb capture --address=192.168.255.255 3000
    $ vdb capture 3000 -a 192.168.255.255
    
  Captures VDIS traffic on port 4000, just the Entity State PDUS from
  entities 1.3.6 and 1.3.7, saves to file 'capture.vdb', prints short
  summary of PDU contents:
  
    $ vdb capture 4000 capture.vdb --pdu-type 1 --ids=1.3.6,1.3.7 --summarize
    $ vdb capture -st 1 -i 1.3.6,1.3.7 4000 capture.vdb
